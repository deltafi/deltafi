## [2.41.0] - 2025-12-06

### Added
- Persona-specific getting started guides: for-operators.md, for-plugin-developers.md, for-core-developers.md
- "Understanding the TUI" section to TUI.md explaining version model and orchestration modes
- Configuration documentation covering config.yaml and site directory customization
- Gradle targets for image scanning:
  - `gradle grype:<project>` - grype scan of project
  - `gradle grypeReport` - summary of all projects
  - `gradle grypeFullReport` - summary of all images running in current local cluster
  - `gradle trivy:<project>` - trivy scan of project
  - `gradle trivyReport` - summary of all projects
  - `gradle trivyFullReport` - summary of all images running in current local cluster
- Documentation now includes the list of system permissions along with their descriptions.
- Add a slice action that slices content with an offset and size 

### Changed
- Simplified README.md and CONTRIBUTING.md to point users to deltafi-docs
- Updated metrics.md to reference VictoriaMetrics instead of Graphite
- Improved cross-linking between plugin documentation guides
- Changed the binary ingress to read the `filename` and `dataSource` from the metadata header if they are not provided in their own headers
- Updated permission descriptions

### Fixed
- UI: Fixed auto-focus on System Properties page
- UI: Fixed badge color and icon on Fleet Dashboard table view
- Fixed an issue where the TUI uses a relative path instead of the users home directory when the `HOME` environment variable is not set
- Removed outdated "coming shortly" claim from errors.md
- Deduplicate scanned containers in scan reports, order alphabetically

### Removed
- Obsolete installation guides: install/compose.md, install/kind.md, kind.md

### Tech-Debt/Refactor
- Updated the ingress docs to match the latest API
- Provide a better error message when the DataSource header/metadata is missing on an ingress request
